Abstract

  This paper presents the use of Artificial Intelligence (AI) in both video game development and graphical processing. 
  It will cover topics such as frame generation, Deep Learning Super Sampling (DLSS) and FidelityFX Super Resolution (FSR) 
  Which help to drastically improve graphical performance by using AI-driven techniques without relying solely on raw hardware performance. 
  This paper will also cover applications of generative AI in Game development analysing its potential benefits and ethical challenges which it presents.

Presentation

DLSS and FSR

The first area we will cover in this paper is DLSS (Deep Learning Super Sampling). DLSS is a
technology developed by Nvidia for Nvidia GPUs (Graphical Processing Units/Graphics card). 
DLSS only works on Nvidia GPUs and does not work on other companies GPUs like AMD FSR
does. DLSS has many “neural rendering technologies” those being: Multi Frame Generation, 
Super Resolution and Deep learning Anti-aliasing.

The first of which we will cover is “Multi Frame Generation”. Multi Frame Generation boosts FPS
(Frames per second) up to three frames rendered per frame by using Artificial intelligence (See Figure three)
predict the next upcoming frames. While this technology can improve frame rate by up to three
times (See Figure Four) it comes with the drawback of latency as the latency of the number of frames you usually
generate remain the same. Another drawback of multi frame generation is the higher the
multiplier is, the less accurate the frames become. Many users report “artificing” (Visual
distortions or anomalies) at the highest multiplier but much less at lower frame generation.

The next “neural rendering” technology we will cover is “Super Resolution”. This is much more 
refined than Frame Generation as it does not suffer from the drawbacks of noticeable latency.
Super resolution boosts the raw power of graphical processing by rendering multiple lower-
resolution images with feedback from previous frames to reconstruct the image but at a much
higher resolution. There are usually multiple settings to this and if the difference is resolution is
small there is not a noticeable difference however, if the images the AI is using is much lower
than the resolution you are aiming to render at it can cause details to be lost. (See Figure Two for 
performance impact)

The final DLSS technology we will talk about in this paper is “Deep Learning Anti-Aliasing”
(DLAA). Anti-Aliasing is a technology that removes jagged edges on computer screens by either
blending colours together or making a colour slowly fade over multiple pixels (See Figure one). 
Anti-aliasing usually takes a lot of processing power to render specially if you
are rendering multiple frames. This makes DLAA a useful technology as it is much lighter on the 
load to the GPU as it uses the same technology that Super Resolution does. Unfortunately,
though, the frames generated by DLAA are not the same as if standard Anti-Aliasing is used as
there are noticeable artifacts on objects if they are moving on each frame similar to an after image.

AMD FidelityFX Super Resolution (FSR) essentially the functions the same as DLSS with the
exception that it can be used on any GPU Nvidia, Intel and AMD. This fills the gap where DLSS
cannot be used as not all Nvidia GPUs (Older models) can use it. DLSS is often considered to be
the better option though, especially if you have a compatible Nvidia GPU.
